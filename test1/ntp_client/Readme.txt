NTP 客户端使用说明
概述
这是一个简单的 NTP (网络时间协议) 客户端实现，可以从指定的 NTP 服务器获取准确的时间，并能够设置本地系统时间。

#文件结构
    main.c - 主程序文件，包含时间设置和显示逻辑

    ntp_client.c - NTP 客户端核心实现

    ntp_client.h - NTP 客户端头文件

#编译方法
    使用以下命令编译程序：

    gcc -o ntp_client main.c ntp_client.c

#运行方式
    1. Ubuntu 常规测试（不设置系统时间）   
    ./ntp_client
    此方式仅显示从 NTP 服务器获取的时间，不会修改系统时间。

    2. 开发板获取时间（需要 root 权限）
    sudo ./ntp_client
    此方式会尝试设置系统时间，需要管理员权限。

#功能说明
    从阿里云 NTP 服务器 (182.92.12.11) 获取网络时间

    支持将获取的时间转换为本地时间并格式化显示

    可选设置系统时间功能（需要 root 权限）

    包含超时处理机制（3秒超时）

#注意事项
    设置系统时间需要 root 权限，普通用户运行会显示权限错误但仍会显示获取的时间

    程序使用 UDP 协议与 NTP 服务器通信，端口为 123

    如果网络连接不稳定，可能会导致获取时间失败或超时

    默认使用的 NTP 服务器是阿里云的，如有需要可以修改代码使用其他 NTP 服务器

#错误处理
    程序包含完善的错误处理机制，会报告以下可能的问题：

    套接字创建失败

    设置套接字选项失败

    无效的 NTP 服务器地址

    发送/接收数据失败

    响应超时

    不完整的 NTP 响应

    设置系统时间权限不足

输出示例
    成功运行时将显示如下格式的时间信息：

    Failed to set system time (may need root privileges): Operation not permitted      ----[未使用管理员权限提示修改系统时间失败]
    NTP time: 2025-09-12 14:25:17
